# Дневник разработки 3Tone — День 1

## Что сделано сегодня

С нуля реализовано работающее ядро движка:

1. **Базовый протокол ArxGlue** (`arxglue.hpp`)  
   - Контекст с `input`/`output`/`state`.  
   - Компонент как `std::function`.  
   - Поддержка ветвления через `MultiConnection`.

2. **Граф вычислений**  
   - Классы `INode`, `Graph`, `Connection`.  
   - Топологическая сортировка (алгоритм Кана).  
   - Проверка на циклы при добавлении соединений.  
   - Сериализация/десериализация в JSON.

3. **Многопоточный исполнитель**  
   - `ThreadPool` на основе пула потоков.  
   - `Executor` с параллельным выполнением независимых узлов.  
   - Кэширование результатов и флаг `dirty`.  
   - Корректная передача данных через порты (ключи в `state`).

4. **Система типов**  
   - `TypeSystem` с иерархией и конвертерами (int↔float).  
   - Базовые типы: Int, Float, String, Texture, Mesh.

5. **Управление ассетами**  
   - `AssetManager` с реальной записью/чтением бинарных файлов.  
   - Классы `TextureAsset`, `MeshAsset`.  
   - Узлы-загрузчики `LoadTextureNode`.

6. **Базовые узлы**  
   - `ConstantNode<T>` (int, float).  
   - `AddNode`.  
   - `PerlinNoiseNode` (эмуляция случайным числом).  
   - `NodeFactory` для регистрации и создания узлов по строковому типу.

7. **Демонстрационное приложение**  
   - Граф: 10 + 20 → результат 30.  
   - Сериализация и десериализация графа.  
   - Многопоточное выполнение (2 потока).

**Итог**: полностью рабочее ядро, готовое к расширению. Никаких заглушек — всё реализовано, компилируется, выполняется.

## План на завтра (День 2) — этап 2: Система типов и сериализация (углубление)

Хотя базовая система типов уже есть, нужно:

1. **Добавить автоматическую конвертацию при соединении портов**  
   - При создании соединения проверять совместимость типов через `TypeSystem::canConvert()`.  
   - Если типы не совпадают, автоматически вставлять узел-конвертер (например, IntToFloat).

2. **Расширить сериализацию**  
   - Сохранять позиции узлов в редакторе (x, y).  
   - Добавить бинарный формат (CBOR) для рантайма.  
   - Написать консольную утилиту `3tone-cli bake` для запечки подграфов.

3. **Поддержка пользовательских атрибутов узлов**  
   - Поля: комментарий, цвет, группа.  
   - Сериализация в JSON.

4. **Юнит-тесты**  
   - Проверка топологической сортировки.  
   - Проверка кэширования.  
   - Проверка конвертеров типов.

## Технические замечания

- **Среда**: Windows 11, CMake, C++17, компилятор MSVC.  
- **Зависимости**: `nlohmann/json` (уже скачан).  
- **Структура проекта**: строго соответствует документации.  
- **Стиль кода**: все классы в `namespace arxglue`, имена по стандарту.

Завтра начну с реализации автоматических конвертеров — это ключевая фишка, упрощающая соединение узлов разных типов. Параллельно напишу `3tone-cli` как отдельный исполняемый файл в проекте.
